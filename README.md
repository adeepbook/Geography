# 地理纵深 (Geo-Depth)

> 一张可点击的世界地图:点开任何一个地方,把它从平面符号展开成一个有纵深、有来历的真实对象。

**核心理念:真实 + 理解。** 它不是查询工具,而是给真正喜欢地理的人"看穿一个地方"用的——既能往下拆(天上 / 地面 / 地下),也能往回看(怎么形成、形成之前是什么)。

## ✨ 特性

- 🗺️ **可点击世界地图**,支持可视化切换多种地图样式
- 📇 **精选地点知识卡**:形成时间轴(可拖动 / 播放)+ 垂直拆解(天 / 地面 / 地下)+ 机制剖面图
- 🤖 **AI 学习助手**:点任意地方,用你自带的大模型 key 辅助分析其自然地理与人文地理(预设模块:地形地貌 / 气候 / 水文 / 地质与演化 / 土壤植被 + 人口聚落 / 经济产业 / 历史文化 / 人地关系 / 土地利用)
- 📝 **学习笔记**:随手收藏 AI 回答与卡片内容,一键导出 Markdown
- 🔑 **全部自带 key(BYOK)**:AI 与高级地图都用你自己的 key,只存在你本机浏览器,不上传任何服务器

## ⚠️ 关于内容真实性(请先读)

本项目把内容分成两类,界面上有明确区分:

- **已核对精选卡**:少数地点,内容经人工 / 资料核对(仍在持续核对中,部分条目标注"待核对")。
- **AI 生成内容**:其余任何地方由大模型即时推测生成,**未经核对、可能有误**,仅供辅助理解,不作权威依据。

地质、地理事实请以权威资料为准;AI 回答请自行核实。

## 🚀 本地运行

前置:已安装 [Node.js](https://nodejs.org)(18 及以上)。

```bash
git clone https://github.com/adeepbook/Geography.git
cd Geography
npm install
npm run dev
```

浏览器打开终端显示的地址(通常 `http://localhost:5173`)。

## 🔧 配置(在应用内"设置"里填,无需改代码)

- **AI 助手**:填 Base URL(默认 `https://api.deepseek.com`,兼容 OpenAI 接口)、你的 API Key、模型 ID。
- **地图样式**:默认的免费无密钥样式开箱即用;想要卫星 / 地形等专业样式,可在设置里填自己的 [MapTiler](https://maptiler.com) 免费 key 解锁。

> 所有 key 仅存在你本机浏览器的 localStorage,不会上传到任何服务器,也不会进入代码仓库。各用各的 key。

## 🧱 技术栈

TypeScript · React · Vite · MapLibre GL JS · 内容为 Git 仓库内的 YAML / Markdown(无数据库、纯前端)。

## 📁 结构

```
content/
  locations/   地点脚本 (YAML)
  processes/   过程机制模块 (Markdown)
src/           前端代码
```

内容与代码解耦:地点脚本通过「模块ID + 参数」引用过程库,知识卡据此渲染;剖面图按事件的机制模块自动绘制。

## 🗃️ 内容状态

每条内容带状态:`起草 / 待核对 / 已核对`。当前精选地点多为起草 / 待核对,欢迎核对与补充。

## 🤝 贡献 / 自己加地点

- **加地点**:在 `content/locations/` 按现有 YAML 格式新增一份脚本(地点ID、坐标、类型标签、一句话、形成时间线、垂直拆解、事实来源、状态)。
- **核对内容**:把核对无误的条目状态改为 `已核对`,并补上 `事实来源`。
